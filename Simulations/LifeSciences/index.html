HS-LS1-1

Construct an explanation based on evidence for how the structure of DNA determines the structure of proteins which carry out the essential functions of life through systems of specialized cells.

  • DNA to Protein Structure and Function - An interactive simulation demonstrating how DNA sequence determines protein structure and specialized cell function. [2026-03-12 12:00:00]

    Evaluation: Investigative Phenomenon | 4.0/5 Stars | 2026-03-12 18:00:00
    • Overview: Functions as a strong investigative phenomenon allowing students to trace how DNA sequence changes cause cascading effects on protein folding and cellular function. It meets Criterion 4 (Investigable Through Practices) by allowing students to directly manipulate the gene sequence. It partially meets Criterion 1 (Cultural/Personal Relevance) through the Sickle Cell scenario, but could be strengthened.
    • Dimensional Evaluation & Evidence Statements: The simulation strongly supports the DCI (LS1.A) and CCC (Structure and Function). It supports the SEP (Constructing Explanations) by providing an "Evidence Log" for students to record data to build their explanation. It supports observable features 1.a, 2.a.i-iv, and 3.a.i-iv by showing how gene sequence (DNA) translates to amino acids, folded protein structure, and ultimate cell function. However, the simulation limits student agency in building the explanation directly within the tool.
    • AI Action Items for Improvement:
      • Improve Relevance (Criterion 1): Add more real-world, highly relevant presets (e.g., cystic fibrosis mutation, BRCA1 mutation) to the "Explore Scenarios" section, providing brief narrative descriptions to increase personal/cultural relevance.
      • Improve SEP (Constructing Explanations): Add a text area at the bottom of the "Evidence Log" titled "Construct Your Explanation" where students can type their final explanation using the data they logged as evidence, directly addressing the prompt.
    • Implementation Checklist for AI Agent:
      • [ ] Add 2-3 new disease-related DNA sequence presets with narrative context.
      • [ ] Add a `<textarea>` element below the Evidence Log table for students to write their explanation.

Develop and use a model to illustrate the hierarchical organization of interacting systems that provide specific functions within multicellular organisms.

  • Hierarchical Organization of Interacting Systems - An interactive model illustrating the hierarchical organization of cells, tissues, organs, and systems in multicellular organisms. [2026-03-12 12:00:00]

    Evaluation: Investigative Phenomenon | 3.5/5 Stars | 2026-03-12 18:00:00
    • Overview: Serves as an investigative phenomenon where students click through levels to see how systems are built. It addresses the concept well but struggles with Criterion 4 (Investigable Through Practices), as students are merely clicking through static levels rather than actively investigating a problem or building the model themselves.
    • Dimensional Evaluation & Evidence Statements: Supports DCI (LS1.A) by showing hierarchical structural organization. Supports CCC (Systems and System Models) by simulating interactions. Weakly supports the SEP (Developing and Using Models) because the model is pre-built; the student does not "develop" it. It covers observable features 1.a (identifies parts) and 2.a.i (functions of systems).     • AI Action Items for Improvement:
      • Fulfill Observable Feature 2.a.ii (System Interactions): Add a feature that visually links the currently viewed system to another system (e.g., show how the Respiratory System provides oxygen to the Circulatory System).       • Improve SEP (Developing Models): Instead of just clicking through pre-set levels, add a "Build Mode" where students must drag and drop components (cells, tissues, organs) into the correct hierarchical order to construct the system themselves.
    • Implementation Checklist for AI Agent:
      • [ ] Implement a visual "Interactions" overlay to show cross-system dependencies.
      • [ ] Create a drag-and-drop "Build Mode" for assembling the hierarchy.
